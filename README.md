# Проект Мяу-мяу-мяу

## Структура Программы

Проект состоит из нескольких файлов, каждый из которых несет свою ответственность:

*   `Program.cs`: Главный класс, точка входа в приложение. Управляет меню и логикой взаимодействия с пользователем.
*   `MenuOption.cs`: Перечисление (`enum`), определяющее пункты меню.
*   `IMeowable.cs`: Интерфейс, который определяет "умение мяукать".
*   `Cat.cs`: Класс `Cat`, реализующий интерфейс `IMeowable`.
*   `Dog.cs`: Класс `Dog`, также реализующий `IMeowable` для демонстрации полиморфизма.
*   `MeowableCounter.cs`: Класс-декоратор, который добавляет функциональность подсчета "мяуканий".
*   `Funs.cs`: Вспомогательный класс со статическими методами для работы с коллекциями `IMeowable`.

## Алгоритм Работы

Работа программы построена на простом цикле в файле `Program.cs`:
1.  **Инициализация**: При запуске создается экземпляр кота по умолчанию (`new Cat("Барсик")`). Этот кот "оборачивается" в `MeowableCounter` для подсчета его действий.
2.  **Главный цикл**: Программа входит в бесконечный цикл `while (true)`.
3.  **Отображение меню**: На каждой итерации цикла пользователю выводится меню с опциями, определенными в `MenuOption.cs`.
4.  **Ввод пользователя**: Программа ожидает, пока пользователь введет номер пункта меню.
5.  **Обработка выбора**: С помощью конструкции `switch` программа определяет, какое действие выполнить в зависимости от выбора пользователя (создать нового кота, заставить его мяукать, выполнить тестовый сценарий и т.д.).
6.  **Выполнение**: Вызываются соответствующие методы классов.
7.  **Возврат к меню**: После выполнения действия цикл начинается заново. Выход из программы происходит при выборе опции "Выход".

## Описание Классов и Шаблонов

### `IMeowable.cs`
*   **Ответственность**: Определяет контракт для всех "мяукающих" существ.
*   **Состав**: Содержит одно объявление метода: `void meow();`. Любой класс, реализующий этот интерфейс, обязан предоставить свою версию этого метода.
*   **Шаблон**: **Интерфейс (Interface)**. Позволяет работать с объектами разных классов (`Cat`, `Dog`) единообразно.

### `Cat.cs`
*   **Ответственность**: Представляет сущность "Кот".
*   **Состав**:
    *   Свойство `Name` для хранения имени.
    *   Метод `meow()`, который выводит в консоль "{Имя}: мяу!".
    *   Перегруженный метод `Meow(int n)`, который вызывает `meow()` `n` раз.
*   **Реализация**: Реализует `IMeowable`.

### `Dog.cs`
*   **Ответственность**: Представляет сущность "Собака", которая пытается мяукать.
*   **Состав**:
    *   Свойство `Name`.
    *   Метод `meow()`, который выводит в консоль "{Имя}: мя... гав! (ну... он пытался...)".
*   **Реализация**: Реализует `IMeowable` для демонстрации **полиморфизма**. Хотя `Dog` и `Cat` — разные классы, благодаря общему интерфейсу мы можем вызвать у них один и тот же метод `meow()` и получить разное поведение.

### `MeowableCounter.cs`
*   **Ответственность**: Добавляет функциональность подсчета вызовов `meow()` к любому `IMeowable` объекту, не изменяя его исходный код.
*   **Состав**:
    *   Хранит ссылку на "обернутый" `IMeowable` объект.
    *   Свойство `Counter` для хранения количества вызовов.
    *   Метод `meow()`, который сначала вызывает `meow()` у обернутого объекта, а затем увеличивает свой счетчик.
*   **Шаблон**: **Декоратор (Decorator)**. Это структурный паттерн, который позволяет динамически добавлять объектам новую функциональность, оборачивая их в полезные "обертки".

### Реализация "Мяукнуть N раз"
Эта функциональность реализована через перегрузку метода в классе `Cat`.

1.  **Определение метода**: В классе `Cat.cs` определен метод `public void Meow(int n)`.
2.  **Логика**: Внутри метода используется цикл `for`, который итерируется `n` раз. На каждой итерации вызывается основной метод `meow()` (без параметров).
3.  **Вызов из `Program.cs`**: Когда пользователь выбирает пункт меню "Мяукнуть N раз", программа запрашивает число `n`, считывает его и затем вызывает `cat.Meow(n)`.
